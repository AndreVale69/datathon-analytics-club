# raw_data

This directory is a placeholder for the challenge dataset.

The actual data is not committed to GitHub and must be downloaded manually from the shared OneDrive source provided by the team or organizers.

Instructions:

1. Download the dataset archive from the shared source.
2. Extract the archive locally.
3. Copy or move the extracted dataset contents into this `raw_data/` directory.
4. Confirm the dataset files are present before starting the API or Docker build.

Notes:

- Do not add public links to the data source in the repository.
- Do not commit the dataset files to Git unless your team explicitly decides to version them.
- Keep credentials and passwords out of the repository and out of tracked files.
